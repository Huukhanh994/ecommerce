Semaphore is a hosted continuous integration and delivery service, built for speed and simplicity. We’re here to make you and your team more productive.

Pipelines, Blocks, Jobs

The basic building block is a pipeline:

Pipelines are made of sequential blocks. Each block may contain a single job, or multiple parallel jobs.

For more advanced use cases, pipelines may also run in parallel and depend on each other.

Promotions

Model your continuous delivery process by connecting pipelines with promotions.

Promotions can be triggered manually or automatically on certain conditions, like pipeline result, Git branch or tag.

There’s no limit in how many pipelines and promotions your workflow can have.

Workflow Builder

Semaphore configuration lives in YAML files in the .semaphore directory of your Git repository.

Further chapters of this tour will show you some snippets of YAML for illustrative purposes. But, we know that writing YAML from scratch is hard.

So Semaphore has a visual Workflow Builder so that you can easily create and edit configuration files for your project:

Agents

Agents specify the hardware and software entry point for your commands.

Hardware. Most projects will be fine with the default e1-standard-2 Linux (2vCPUs, 4GB RAM) and a1-standard-4 Mac (4 vCPUs, 8GB RAM) machine types.

Software. You can use Semaphore's Linux or macOS VMs with conveniently preinstalled software:

agent:
    machine:
        type: e1-standard-2
        os_image: ubuntu1804
agent:
    machine:
        type: a1-standard-4
        os_image: macos-mojave

Or you can run your own environment, defined with one or multiple Docker containers:

agent:
    machine:
    type: e1-standard-2
    containers:
        - name: main
          image: semaphoreci/ruby:2.6.1
        - name: cache
          image: redis:5.0

Caching

Every job runs in a fresh, isolated virtual machine. Caching is a way to reuse project dependencies across pipeline stages

cache restore
npm install
bundle install --path .bundle
cache store

Cache is scoped per project and can store up to 9GB of files.

Artifacts

Artifacts is permanent storage for files that you want to deliver to users or reference yourself later. For example:

  • A binary of your app
  • A build of your static website
  • Screenshots and logs of failed tests

Semaphore also provides a Docker registry that’s private to your organization, to save you time and network bandwidth.

Secrets

Secrets help your CI/CD pipelines talk to the outside world.

It's a secure way of storing private information like API keys, or configuration files — things that shouldn't be committed to source control.

Secrets are scoped to organization. Make them available by listing them within a pipeline or block:

blocks:
  - name: Deploy
    task:
    secrets:
      - name: deploy-secrets
    jobs:
        - name: Push to production
        commands:
        - echo "$CLOUD_ACCESS_KEY_ID"

Debugging

Sometimes your pipelines will fail, and at first you won't know why.

In that case debugging via SSH is a lifesaver.

Every job has a "Debug" button with instructions for installing the Semaphore CLI and SSH-ing into your job.

The CLI can do many more things, but we'll let you explore that later.
